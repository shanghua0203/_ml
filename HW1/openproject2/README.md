# TSP 視覺化系統 (TSP Visualizer)

使用 PyGame 實現的旅行推銷員問題 (Traveling Salesman Problem) 視覺化工具，支援即時動畫顯示尋找最短路徑的過程。

## 功能特色

- **即時路徑動畫**：可看到演算法嘗試新路徑的過程
- **收斂曲線圖**：圖表顯示最佳距離隨迭代次數的變化
- **多種演算法**：支援爬山法、暴力法、模擬退火
- **互動式操作**：
  - 滑鼠點擊放置/刪除城市
  - 速度滑桿控制動畫速度
  - 播放/暫停/重置控制

## 安裝與執行


## 使用說明

### 控制面板

| 按鈕 | 功能 |
|------|------|
| 隨機城市 | 隨機生成 8 個城市 |
| 清除全部 | 清除所有城市 |
| 開始 | 開始執行演算法 |
| 暫停 | 暫停執行 |
| 重置 | 重置到初始狀態 |

### 速度控制

使用「速度」滑桿調整動畫速度：
- 數值越大，演算法執行越快
- 建議：小城市用慢速觀察過程，大城市用高速加速搜尋

### 步數限制

使用「步數限制」滑桿設定最大執行步數：
- 設為 0 表示無限制
- 到達步數上限後自動停止

### 演算法選擇

點擊演算法按鈕即可切換：
- **爬山法**：貪心搜尋，速度快但可能陷入局部最優
- **暴力法**：窮舉所有排列，確保找到最優解（適合城市數 ≤ 8）
- **模擬退火**：機率性接受較差解，有機會跳出局部最優

### 視覺化元素

| 元素 | 說明 |
|------|------|
| 灰色圓點 | 城市節點（帶編號） |
| 白色線 | 當前路徑 |
| 藍色虛線 | 正在比較的新路徑 |
| 綠色粗線 | 目前找到的最佳路徑 |
| 底部圖表 | 收斂曲線（最佳距離 vs 迭代次數） |

### 滑鼠操作

| 操作 | 功能 |
|------|------|
| 左鍵點擊畫布 | 新增城市 |
| 右鍵點擊城市 | 刪除城市 |

## 技術實作

### 演算法設計

所有演算法都實作為「生成器模式」(Generator Pattern)：
- 每一步 `step()` 只執行固定次數的迭代
- 透過 `get_state()` 返回當前狀態
- 可隨時暫停、恢復執行

### 繪圖架構

- **Renderer 類別**：負責所有 PyGame 繪圖操作
- **TSPVisualizer 類別**：管理應用程式狀態和事件處理
- 60 FPS 遊戲循環確保流暢的動畫效果

### 參數配置

可在 `config.py` 中調整：
```python
WINDOW_DEFAULT_SIZE = (1200, 800)  # 預設視窗大小
CONTROL_PANEL_WIDTH = 260          # 控制面板寬度
CHART_HEIGHT = 150                # 收斂圖高度
CITY_RADIUS = 15                   # 城市圓點半徑
DEFAULT_CITY_COUNT = 8             # 隨機生成城市數量
CHART_MAX_POINTS = 200             # 收斂圖最大資料點數
```
